ДЗ: Свой сетевой фреймворк.

Написать свой тестовый фреймворк.
Поддержать свои аннотации @Test, @Before, @After.
Запускать вызовом статического метода с именем класса с тестами.

Т.е. надо сделать:
1) создать три аннотации - @Test, @Before, @After.
2) Создать класс-тест, в котором будут методы, отмеченные аннотациями.
3) Создать "запускалку теста". На вход она должна получать имя класса с тестами, в
   котором следует найти и запустить методы отмеченные аннотациями из пункта 1.
4) Алгоритм запуска должен быть следующий::
   метод(ы) Before
   текущий метод Test
   метод(ы) After
   для каждой такой "тройки" надо создать СВОЙ объект класса-теста.
5) Исключение в одном тесте не должно прерывать весь процесс тестирования.
6) На основании возникших во время тестирования исключений вывести статистику
   выполнения тестов (сколько прошло успешно, сколько упало, сколько было всего)

** План действий для реализации Класса "запускалку теста".
3.1 Reflection для получения информации о классе - Class.forName()
3.2 Поиск аннотированных методов - getDeclaredMethods() + isAnnotationPresent()
3.3 Создание нового объекта для каждого теста - newInstance()
3.4 Вызов методов в правильном порядке - Before → Test → After
3.5 Обработка исключений - ошибка в одном тесте не ломает другие
3.6 Статистика - подсчет успешных/упавших тестов
и запускаем ч/з Main
